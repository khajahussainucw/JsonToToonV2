What is JSON Escape / Unescape?

Need to escape JSON strings for HTML or JavaScript? Or unescape them back? This tool does both. It escapes special characters in your JSON (JavaScript Object Notation) strings to make them safe, or unescapes them to restore the original characters.

As MDN Web Docs explains, JSON strings must escape certain characters like quotes, backslashes, and control characters. Our tool handles all standard escape sequences from the RFC 8259 JSON specification, so your strings are properly formatted.

Everything happens in your browser – your data stays private. Switch between escape and unescape modes with one click.

Common Questions

Is my JSON data stored?

Nope. Everything happens right in your browser. Your data never leaves your computer – we don't even see it.

What characters get escaped?

Standard JSON escape sequences include: quotes (\"), backslashes (\\), newlines (\n), tabs (\t), carriage returns (\r), and Unicode characters (\uXXXX).

What's the difference from URL encoding?

JSON escaping is different from URL encoding. JSON escaping uses backslash sequences (like \n), while URL encoding uses percent sequences (like %20). Use JSON escaping for JSON strings.

Can I escape entire JSON objects?

Yep. You can escape entire JSON objects or just JSON strings. We process whatever you paste, converting special characters to escape sequences.

Does unescaping validate JSON?

Unescaping converts escape sequences back to characters. It doesn't validate JSON syntax – use our JSON Validator for that.

When should I escape JSON?

Escape JSON when embedding it in HTML attributes, JavaScript strings, or anywhere special characters might cause syntax errors. Unescape when you receive escaped JSON and need the original characters.
